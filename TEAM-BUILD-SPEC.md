# カバルドン軸チームビルド仕様書

## 概要

pokechamdb TOP30 シングルメタに対して、**カバルドン先発固定**の最強3体基本選出を逆算で導出するプロジェクト。

### 前提条件
- データソース: pokechamdb.com 2026/04 シングル TOP30
- 計算エンジン: Pokemon Champions ダメージ計算機 (`src/`)
- フィールド: `Singles, Sand` (カバルドンのすなおこし)
- 対戦形式: 3v3 シングル

## Phase 1: カバルドン最適育成の導出

### 1.1 役割定義

カバルドンの先発としての行動パターン:

```
Turn 1: ステルスロック (※相手を地震で確1にできる場合のみ地震)
Turn 2: あくび or 地震
Turn 3+: つるぎのまい / あくび / 地震 / ほえる
```

**重要な帰結**: カバルドンは「2耐えしてステロ+あくび」が仕事。確1で落とせる相手以外、攻撃性格のメリットはない。

### 1.2 性格検討

| 性格 | 補正 | Atk | Def | SpD | 評価 |
|------|------|-----|-----|-----|------|
| いじっぱり (Adamant) | +A -C | 145 | 154 | 100 | 確1対象: ルカリオのみ。防御が薄い |
| わんぱく (Impish) | +B -C | 132 | 169 | 110 | **物理耐久↑、特殊もSP18で補強** |
| しんちょう (Careful) | +D -C | 132 | 154 | 121 | 特殊耐久↑だが物理が手薄 |

**結論**: いじっぱりで確1にできるのはルカリオ1体のみ。防御性格の方が耐える範囲が広がる。

#### 集計ルール
- **ポケモン単位**: 各相手の最強技1つで判定 (技単位ではない)
- **採用率20%以上の攻撃技のみ対象** — 20%未満の技はケアしない
- **乱1以内**: koN=1 (確1 or 乱1)
- **乱2以内**: koN≤2 (確1〜乱2)
- **継続技のみ**: 流星群・オーバーヒート除外 (C-2ペナルティで1回限り)
- **砂下ソーラービーム除外**: メガリザードンYの日照りはすなおこしに上書き → ソーラービーム溜め必要

#### オボンのみ (Sitrus Berry) 発動ロジック

- **発動条件**: HP ≤ 50% (floor(maxHP/2)) になった時点で自動発動
- **回復量**: floor(maxHP/4) = 約25%
- **確2の閾値**: オボン込みで確2にするにはダメージ ≥ 62.5%/発 が必要
  - 1発目で50%以上削られる → HP≤50% → オボン発動 → HP回復
  - 2発目でKOするには (ダメージ×2) ≥ 125% → 1発あたり62.5%以上
- **⚠ straddle警告**: 乱数が50%を跨ぐ場合 (例: 48-55%)、低乱数ではオボン不発動で
  2発KOの可能性あり → 乱2(⚠)として警告表示

#### 単発技の扱い
- **流星群 (Draco Meteor)**: C-2段階ダウン → 2回目は威力激減。確1でない限り脅威度低
- **オーバーヒート**: 同上
- **ソーラービーム**: 砂下ではチャージ必要 (メガリザードンYの日照りはすなおこしに上書きされる)

## Phase 3: 後続2体の最適パートナー導出

### 3.0 Phase 3 旧評価の問題点 (破棄)

旧 §3.1〜3.7 は以下の致命的な欠陥があり、**結論を含め全面破棄**:

1. **速度無視の Handles 判定**: 「確2以内 + 相手の確1を耐える」だけで先手/後手を区別しなかった。後手確2 vs 相手先手確2 は実際には負けるのに "Handles" にカウント
2. **SR と砂ダメの混同**: ステルスロック (交代出し時のみ1回) と砂ダメ (毎ターン1/16) を合算して "chip" として確定数に常時反映していた。対面中の相手にSRは効かない
3. **マルチスケイル無効化の過小評価**: 砂でマルチスケイルが剥がれることは計算済みだったが、メガカイリューが1位→5位に転落する重大な影響を見落としていた
4. **車輪の再発明**: 既存の `matchupValue()` (速度考慮済み連続スコア) を使わず、独自に二値分類を実装してバグを埋め込んだ

### 3.1 チップダメージの正しい区分

| ダメージ源 | タイミング | 量 | デフォルト考慮 |
|-----------|-----------|-----|-------------|
| **砂ダメージ** | 毎ターン終了時 | 最大HPの1/16 (6.25%) | **○ 常時** |
| **ステルスロック** | 交代で出てきた時に1回 | 最大HPの1/8 × いわ相性 | **△ オプション** |

- 砂免除: いわ・じめん・はがねタイプ
- SR免除: 場に出ている対面の相手 (交代していない相手にはSRは関係ない)
- **デフォルト評価は砂のみ**。SR込みは「カバルドンが SR+あくび/ほえるで相手の控え全員にSRを踏ませた場合」のオプション

## Phase 4: 2体縦突破ペアの選定

### 4.1 Phase 3 → Phase 4 への根本的転換

**Phase 3 までの誤り**: 「3体の合計カバレッジで TOP30 を全カバーすればよい」
**Phase 4 の正しい設計思想**: カバルドンは SR+あくび/ほえるで退場し、**残り2体で相手3体を砂+SR込みで縦に突破する**

#### カバルドン先発の試合展開

```
Turn 1: カバルドン → ステルスロック (砂発動)
Turn 2: あくび → 相手は居座るか交代を迫られる
Turn 3: ほえる/じしん → 相手の控え全員にSR強制 or 削り
...
カバルドン退場 → 残り2体 vs 相手3体 (全員SR+砂ダメ蓄積済み)
```

**帰結**: パートナー2体に求められるのは「個別対面の勝ち数」ではなく、**「3体を連戦で倒し切れる継戦能力」**。

### 4.2 評価手法

#### スクリプト: `hippo-partner-pair.ts`

`evaluate3v3()` を使用して「カバルドン + P1 + P2」vs 「相手3体」を227パターン評価。

**相手チーム生成方法**:
- Method 1: TOP30 各エースの最多パートナー2体で構成
- Method 2: TOP30 から種族・メガ条項を満たす3体組合せ全列挙
- 合計227チーム

**候補フィルタ**: overallScore ≥ 35 のプールメンバー → 282体

## 意思決定の経緯

### ADR-HB-001: 防御性格 > 攻撃性格
- **決定**: わんぱく (Impish) を採用、いじっぱり (Adamant) を不採用
- **理由**: いじっぱりで確1にできるのはルカリオ1体のみ。カバルドンの役割は「2耐え→ステロ+あくび」であり、確1でない限り攻撃力にメリットなし。防御補正はB169→物理耐久が大幅改善。

### ADR-HB-002: わんぱく B6/D28 を採用
- **決定**: わんぱく (Impish, +B) / SP: H32/B6/D28 を採用
- **理由**: 網羅探索(わんぱく/しんちょう × B0-32 step2 + H削減)の結果、継続技で乱2以内=10体が最少ライン。同スコアのビルド3つを個別検討:
  - わんぱく B6/D28: ウォッシュロトム確2、イダイトウ乱1(25%) → **採用**
  - わんぱく B14/D20: ウォッシュロトム乱1(38%) → 不採用 (ロトムは使用率#15で頻出)
  - しんちょう B6/D28: イダイトウ乱1(81%) → 不採用 (75%耐えない)
- **H削減の検証**: H30/28/26でB/Dを増やすパターンも検証。HP減のデメリットがB/D増を上回り全て悪化。

### ADR-HB-003: A0 (攻撃SP無振り)
- **決定**: Atk SPを0に固定、66ポイント全てをH/B/Dに配分
- **理由**: A0→A2でAtk132→134 (+1.5%)。閾値を超える対面なし。一方で防御-2は乱2ボーダーを悪化させるリスクあり。

### ADR-HB-004: 単発技の分離集計

- **決定**: 流星群・オーバーヒートは「継続技」と分離して集計
- **理由**: C-2段階ダウンにより実質1回限り。確1でない限り2発目の威力が激減するため、乱2の脅威度としてカウントするのは不適切。

### ADR-HB-005: 採用率閾値20% + オボン込みKO計算

- **決定**: 採用率20%未満の攻撃技は脅威度評価から除外。KO判定はオボンのみの回復を正しく反映する。
- **理由**:
  - 旧閾値(5%)ではとびひざげり(7%)等の超マイナー技が混入し、実戦で遭遇しない脅威を過大評価していた。
  - 旧KO判定 (`koChance()`) はオボンのみの回復を一切考慮せず、50-62.5%のダメージを「確2」と誤判定していた。
  - オボン込み確2の正しい閾値: 1発あたり ≥62.5% (2発合計 ≥125%)。
  - ダメージ乱数が50%を跨ぐ場合 (例: 48-55%) は、低乱数時にオボン不発動→2発KOの可能性があるため「乱2(⚠)」として警告。
- **影響**: わんぱく H32/B16/D18 の乱2以内: 旧18/29 → 修正後16/29 (継続技のみ: 旧15 → 修正後13)。

### ADR-HB-006: C-drop技の偽確2排除 (matchupValue修正)

- **決定**: りゅうせいぐん・オーバーヒート・リーフストーム・フルールカノン・サイコブーストで koN≥2 の場合、`matchupValue()` はスコア0を返す
- **理由**: C-2段階ダウンにより2発目の威力が半減。matrix上「確2」でも実際には2発KO不可能。これが選出シミュレーションで「先発りゅうせいぐん確2で殴ってくる」という非現実的な想定を生んでいた
- **実装**:
  - `DamageMatrixEntry` に `isStatDrop?: boolean` フラグ追加
  - `team-matchup-core.ts` に `STAT_DROP_MOVES` 定数 + `matchupValue()` に早期リターン
  - `team-matchup.ts` の matrix 生成で `isStatDrop` フラグを設定
  - 既存matrixデータは `STAT_DROP_MOVES.has(entry.bestMove)` でフォールバック
- **影響**: 96100マッチアップ中2825件 (2.9%) が偽確2→スコア0に修正。パートナーペア評価の結果は変化なし (メガフラエッテ+イダイトウ♀が引き続き1位)

### ADR-HB-007: スカーフ速度の poolSpeeds 反映

- **決定**: `team-matchup.ts` の poolSpeeds と matrix 生成で primaryBuild の実効速度（スカーフ込み）を使用する
- **理由**: `singlesScores.speedStat` はビルド加重平均のため、スカーフ補正が希釈されていた。イダイトウ♀(S=205)がスカーフ込みS=307のはずだが205として比較されていた
- **実装**:
  - `buildSpeed()` ヘルパー追加: `calcStat()` + スカーフ判定
  - matrix 生成 (L281-282): `singlesScores.speedStat` → `buildSpeed(name, build)`
  - poolSpeeds 設定 (L2387): 同上
  - pool 出力の speedStat フィールド: 同上
- **影響**: スカーフ持ち2体（イダイトウ♀、ヒスイバクフーン）の matchupValue が正しく先手評価に。TOP30にスカーフ持ちがいないため相手側への影響なし。味方イダイトウ♀のスコアが上昇（過小評価の修正）
